# ISS-003: Hardcoded kernel parameters in setup_vm_components

Created: 2026-03-31
Status: RESOLVED
Severity: MEDIUM
Source: Full codebase review (2026-03-31)

## Description

`src/crosvm/sys/macos.rs` lines 126-132 inject several hardcoded kernel command-line parameters into `extra_kernel_params` that were added during development:

```rust
params.push("earlycon=uart8250,mmio,0x3f8".to_string());
params.push("earlycon=uart8250,mmio32,0x3f8".to_string());
params.push("panic=30".to_string());
params.push("keep_bootcon".to_string());
params.push("rdinit=/init".to_string());
```

## Analysis

| Parameter | Purpose | Should Keep? |
|-----------|---------|--------------|
| `earlycon=uart8250,mmio,0x3f8` | Enable earlycon for early serial output | **YES** — but should come from serial_parameters.earlycon, not hardcoded |
| `earlycon=uart8250,mmio32,0x3f8` | Duplicate earlycon with mmio32 variant | **NO** — redundant, the first one works |
| `panic=30` | Wait 30s before reboot on panic | **NO** — debug convenience, overrides defconfig `panic=-1` |
| `keep_bootcon` | Keep earlycon active after console takeover | **MAYBE** — useful for debugging but adds overhead |
| `rdinit=/init` | Force initramfs init path | **NO** — should not be hardcoded, kernel finds /init automatically |

## Impact

- User cannot control these parameters via `--params` because they're injected unconditionally
- `panic=30` conflicts with defconfig `panic=-1` (immediate reboot)
- Duplicate earlycon parameter wastes cmdline space
- `rdinit=/init` is unnecessary — kernel already checks /init by default when initramfs is present

## Recommended Fix

1. Remove `earlycon=uart8250,mmio32,0x3f8` (duplicate)
2. Remove `panic=30` (let defconfig control)
3. Remove `keep_bootcon` (debug only)
4. Remove `rdinit=/init` (unnecessary)
5. Keep single `earlycon=uart8250,mmio,0x3f8` but derive from `serial_parameters.earlycon` flag instead of hardcoding

The earlycon parameter should be generated by `get_serial_cmdline` when `serial_parameters[COM1].earlycon = true`, which is already set in the code at line 215.
